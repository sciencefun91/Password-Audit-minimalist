Introduction

I chose to dedicate my final project to the theme
of password security—an aspect often underestimated
but fundamental in protecting computer systems.

Passwords are indeed the primary authentication
mechanism and, when managed inadequately, become
one of the most exposed points to attacks.

Improving their use thus means significantly
reducing the risk surface for both users and services.

With this in mind, the developed website integrates
two complementary tools designed to be clear,
reliable, and technically transparent: a function
to verify credential exposure via the Have I Been
Pwned API, and a configurable secure password generator.

Both aim to promote more conscious and robust
choices in the daily management of one’s digital credentials.

Tools

1. Data Breach Check

Instantly verify if your email appears in known data breaches using the Have I Been Pwned API.

2. Secure Password Generator

Create strong, customizable passwords with options for length and complexity.

Common Password Mistakes to Avoid

  1. Using extremely simple passwords or those found in public dictionaries.
  2. Reusing the same password across multiple services.
  3. Not enabling two-factor authentication (2FA) when available.
  4. Choosing passwords based on personal information that's easily discoverable.
  5. Using passwords that are too short, making them vulnerable to brute-force attacks.
  6. Failing to change passwords after breaches or prolonged use.
  7. Storing credentials in insecure or unencrypted locations.
  8. Sharing passwords through insecure channels (SMS, email, unencrypted messaging).
  9. Adopting predictable patterns like incremental numbers or common suffixes.
  10. Not using a password manager, which leads to password reuse and weak credentials.

Conclusion

This project aims to provide both educational and practical contributions to password security, integrating real tools and fundamental cybersecurity concepts into a simple, transparent, and fully understandable web application.

Following CS50's philosophy, the entire work prioritizes active learning, code clarity, and incremental feature development, with the goal of uniting theory, implementation, and end-user awareness.
